내부 검수 도구 열기 (관리자 전용)
이 도구는 내부 운영자용입니다. 검수 큐 등록은 게시가 아니며, 기존 판정 결과(final_decision · policy_confidence · verification_card)를 변경하지 않습니다. 화면에 보이는 것은 인증이 아니라 단순 표시이며, 실제 보호는 서버의 REVIEW_API_ENABLED 설정과 X-Review-Token 헤더로만 이루어집니다. 여기서 발행되는 결과는 없습니다 (게시가 아님).
서버 백엔드의 리뷰 API에 연결된 내부 검수 도구입니다. 토큰을 보유한 운영자만 사용할 수 있으며, 검증 판정 로직과 공개 화면에는 영향을 주지 않습니다. 사람 검토가 필요한 AI 초안을 모아 검수 큐에 등록하기 위한 내부 작업 영역이며, 이 화면에서 발행되는 결과는 없습니다.
토큰은 브라우저 세션 저장소에만 보관되며 서버 호출 시 X-Review-Token 헤더로만 전송됩니다. 코드, 로그, URL에는 포함되지 않습니다.

분석 결과 → 검수 큐 등록

현재 화면에 표시 중인 분석 결과를 서버 검수 큐에 사람 검토 필요 상태로 등록합니다. 이 동작은 게시가 아니며, final_decision / policy_confidence / verification_card 값은 변경되지 않습니다. 같은 결과를 다시 등록하면 기존 검수 작업이 그대로 사용됩니다.

사람 검토됨 승격 (관리자 전용)

저장된 분석 결과에 human_reviewed_at를 설정/해제합니다. 이는 게시가 아니며 verdict / review_status / 신뢰도 값은 변경되지 않습니다. 비워 두면 현재 화면에 열린 결과의 id를 사용하고, 직접 id를 입력하면 그 결과에 적용됩니다. 승격하면 검증 카드에 "사람 검토됨" 배지가 표시되고, 해제하면 사라집니다.
토큰을 적용하면 서버 검수 큐를 불러옵니다.

How We Verify

        
검증 방법

        

          정책 AI 뉴스 검증 플랫폼은 기사 속 정책·경제 주장을 근거 기반으로 검증 가능성을 평가합니다.
          이 서비스는 단정적인 최종 판정자가 아니라, 공식 출처와 관련 자료를 추적해 신뢰도와 남은 불확실성을 투명하게 보여주는 도구입니다.
        

        

          AI 결과는 사람 검토 전 초안입니다. 공식 상세문서, 의미 매칭, 반박 가능성, 시점과 대상이 충분히 확인되지 않으면
          관찰 또는 사람 검토 필요로 표시합니다.
        

      


      

        

          
무엇을 검증하나요?

          
    
            
  • 정부·공공기관의 정책 발표, 규제, 지원, 조사, 제도 변경처럼 출처와 문서로 확인 가능한 주장
    • 
            
  • 금액, 비율, 대상, 시행 시점, 기관명처럼 원문 대조가 가능한 구체 정보
    • 
            
  • 단순 전망, 의견, 정치적 평가, 익명 제보처럼 독립 근거가 부족한 내용은 확정 판정하지 않습니다.
    • 
          

        

        

          
AI는 무엇을 하나요?

          
    
            
  • 기사 제목과 본문에서 검증 가능한 핵심 주장을 추출합니다.
    • 
            
  • 공식기관 후보, 뉴스 원문, 관련 자료를 정리하고 출처 신뢰도와 의미 매칭을 분류합니다.
    • 
            
  • 반박·모순 가능성과 위험 신호를 표시하지만 최종 사실 판정자는 아닙니다.
    • 
          

        

      


      

        
출처 우선순위

        
    
          
  1. 법령, 고시, 공식 보도자료, 정부·공공기관 원문
    2. 
          
  2. 공공기관 보고서, 공식 통계, 감독기관 자료
    3. 
          
  3. 주요 언론 보도
    4. 
          
  4. 전문기관·연구기관 자료
    5. 
          
  5. 블로그, 커뮤니티, SNS, 익명 캡처
    6. 
        

        

          뉴스 원문은 분석 대상일 수 있지만 공식 검증 근거와는 별도로 취급합니다.
          공식 후보가 있어도 상세 본문에서 기사 핵심 주장과 직접 연결되는 문장을 확인하기 전에는 확정 근거로 보지 않습니다.
        

      


      

        
현재 표시되는 검증 상태의 의미

        

          

            공식 근거 상태: 공식 후보만 있음
            공식기관 또는 관련 기관 후보는 발견했지만, 기사 핵심 주장과 직접 일치하는 공식 상세문서가 아직 확인되지 않은 상태입니다.
          

          

            공식 상세문서 상태: 공식기관 후보는 있으나 상세 본문 미확인
            공식기관 후보는 있으나, 상세 페이지 본문이나 원문 문서에서 핵심 주장과 직접 일치하는 문장을 확인하지 못한 상태입니다.
          

          

            의미 매칭 상태: 의미 매칭 근거 부족
            기사 주장과 수집된 공식·관련 자료 사이의 의미적 연결이 충분하지 않아 자동 확정 판정을 내리기 어려운 상태입니다.
          

          

            반박/모순 상태: 일부 상충 가능성이 있으나 같은 시점과 대상인지 추가 확인이 필요합니다.
            상충 가능성이 있는 자료 또는 신호가 있지만, 같은 정책, 같은 시점, 같은 대상에 대한 반박인지 사람 검토가 필요한 상태입니다.
          

          

            사람 검토 필요
            AI가 자동으로 확정하기 어려운 불확실성이 남아 있어 사람이 원문, 공식문서, 시점, 대상, 맥락을 확인해야 하는 상태입니다.
          

        

      


      

        

          
경고 단계와 신뢰도

          
    
            
  • 낮음(LOW): 현재 수집된 정보 기준으로 즉각적인 높은 정책 리스크 신호는 낮습니다.
    • 
            
  • 관찰(WATCH): 정책 영향 가능성은 있으나 공식 근거, 맥락, 반박 가능성 등 추가 확인이 필요합니다.
    • 
            
  • 주의(CAUTION)·높음(HIGH)·심각(CRITICAL): 더 강한 정책 영향 또는 위험 신호가 있을 수 있으나 최종 판단은 근거와 사람 검토가 필요합니다.
    • 
          

          
신뢰도 점수는 수집된 근거의 품질, 공식 출처 연결성, 의미 매칭, 반박 가능성 등을 종합한 참고 지표입니다.
